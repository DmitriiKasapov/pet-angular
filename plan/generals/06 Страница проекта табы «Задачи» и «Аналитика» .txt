## Шаг 6. Страница проекта: табы «Задачи» и «Аналитика»

Нужно реализовать отдельную страницу проекта, на которой пользователь сможет работать с задачами конкретного проекта и смотреть аналитику по нему.

Маршрут страницы:

`/projects/:id`

Эта страница должна быть одной из основных в приложении и показывать, что проект умеет не только хранить данные, но и давать понятный обзор по задачам и времени.

---

## Назначение страницы

Страница проекта нужна для:

- просмотра информации о конкретном проекте
- просмотра задач этого проекта
- создания новых задач внутри проекта
- перехода в отдельную задачу
- просмотра аналитики по времени внутри проекта

---

## Общая структура страницы

На странице нужно сделать два таба:

- `Задачи`
- `Аналитика`

Переключение между табами должно быть простым и локальным, без отдельных маршрутов внутри проекта.

---

## Верхняя часть страницы

В верхней части страницы нужно показать основную информацию о проекте.

### Что должно быть видно

- название проекта
- код проекта
- описание, если есть
- цвет проекта
- общее количество задач
- общее потраченное время по проекту

### Важно

Общее время проекта должно вычисляться автоматически на основе worklog-записей по всем задачам этого проекта.

---

## Таб 1. Задачи

Этот таб нужен для повседневной работы с задачами проекта.

### Что должно быть внутри

- строка поиска по задачам проекта
- список задач проекта
- кнопка создания новой задачи
- пустое состояние, если задач пока нет

---

## Поиск внутри проекта

В табе задач нужен локальный поиск по задачам этого проекта.

### По каким полям должен работать поиск

- код задачи
- название задачи

### Как должен работать поиск

- локально
- без backend
- сразу при вводе
- без учёта регистра

---

## Список задач проекта

Каждая задача должна отображаться как отдельная строка или карточка.

### Что должно быть видно по задаче

- код задачи
- название задачи
- статус
- потраченное время
- estimated hours, если указано
- кнопка или ссылка для перехода в задачу

### Цветовая индикация статуса

Нужно визуально выделять статус задачи:

- `planning`
- `progress`
- `done`

Статус должен быть заметен сразу, без необходимости открывать задачу.

---

## Создание задачи

На странице проекта должна быть возможность создать новую задачу внутри текущего проекта.

### Минимальные поля задачи

- `title`
- `description` — опционально
- `status`
- `estimatedHours` — опционально

### Генерация кода задачи

Код задачи должен формироваться автоматически на основе кода проекта.

Пример:
- проект `PRO1`
- задачи: `PRO1-001`, `PRO1-002`, `PRO1-003`

### Требования

- номер задачи должен быть последовательным внутри проекта
- после создания задача должна сразу появляться в списке
- задача должна сохраняться в `localStorage`

---

## Переход в задачу

При клике на задачу нужно переходить на страницу:

`/tasks/:id`

---

## Пустое состояние таба задач

Если у проекта ещё нет задач, нужно показать:

- понятный текст
- кнопку создания первой задачи

---

## Таб 2. Аналитика

Этот таб нужен для просмотра суммарного времени по проекту и его задачам.

### Что должно быть внутри

- общее время по проекту
- список задач проекта
- количество часов по каждой задаче

### Как должна выглядеть аналитика

Достаточно простой таблицы или списка.

Для каждой задачи показать:

- код задачи
- название задачи
- статус
- суммарное количество часов

### Дополнительно допустимо

Можно отсортировать задачи по убыванию потраченного времени, чтобы сразу было видно, на что ушло больше всего часов.

---

## Что должно вычисляться на странице проекта

Для текущего проекта нужно вычислять:

- количество задач
- общее время проекта
- потраченное время по каждой задаче

Эти вычисления не должны выполняться прямо в шаблоне.  
Их нужно вынести в сервис или подготовить во view-model страницы.

---

## Загрузка страницы

При открытии `/projects/:id` нужно:

- получить проект по `id`
- загрузить его задачи
- загрузить связанные worklog-записи через задачи
- подготовить данные для таба задач
- подготовить данные для таба аналитики

---

## Поведение, если проект не найден

Если проект по `id` не найден, нужно показать корректное пустое или error-состояние.

### Что допустимо

- сообщение “Проект не найден”
- кнопка возврата в каталог проектов

---

## Что не нужно делать на этом шаге

- не добавлять редактирование проекта
- не добавлять удаление проекта
- не добавлять сложные фильтры по дате
- не добавлять графики
- не делать отдельные вложенные маршруты для табов
- не смешивать weekly analytics со страницей проекта

---

## Ожидаемый результат шага

После завершения этого шага должно быть реализовано:

- страница `/projects/:id`
- верхний блок с информацией о проекте
- таб `Задачи`
- поиск по задачам проекта
- создание новой задачи
- показ статусов задач
- показ потраченного времени по задачам
- переход на страницу задачи
- таб `Аналитика`
- показ общего времени по проекту
- показ времени по каждой задаче
