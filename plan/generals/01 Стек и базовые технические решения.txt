## Шаг 1. Стек и базовые технические решения

Нужно создать современный Angular MVP-проект, который не стыдно показать на собеседовании.  
Проект должен выглядеть актуально по стеку, но при этом оставаться небольшим и реалистичным по объёму.

### Основной стек

- Angular
- TypeScript
- Tailwind CSS
- Angular CDK DragDrop
- Angular Router
- Reactive Forms
- localStorage
- seed-данные

### Требования к стеку

- использовать современный Angular-подход без устаревшей архитектуры
- использовать standalone components
- не использовать NgModule как основу архитектуры
- использовать Angular Router для страниц приложения
- использовать Reactive Forms для всех форм
- использовать Angular CDK DragDrop для недельной доски
- использовать Tailwind CSS для всего UI
- не использовать Angular Material
- не использовать NgRx
- не использовать backend
- не использовать реальную БД
- не использовать SSR

### Подход к хранению данных

Для этого MVP не нужно реализовывать настоящий backend или полноценную базу данных.

Нужно использовать следующий подход:

- начальные демо-данные хранятся в seed-файлах
- при первом запуске приложения seed-данные копируются в `localStorage`
- после инициализации все операции создания, редактирования и удаления работают только через `localStorage`

### Зачем это нужно

- приложение сразу имеет реалистичные стартовые данные для демонстрации
- проект выглядит живым уже после первого запуска
- это позволяет уложить разработку в 1–2 дня
- фокус остаётся на Angular, структуре приложения и UI-логике, а не на backend

### Требования к хранению данных

- если `localStorage` пустой, нужно инициализировать его из seed-данных
- если в `localStorage` уже есть данные, они становятся основным источником истины
- логика хранения должна быть вынесена в отдельный сервис
- seed-данные должны включать несколько проектов, задач и записей worklog для демонстрации
- в рамках MVP не использовать API, сервер или реальную базу данных

### Общие ограничения MVP

- один пользователь
- без авторизации
- без subtasks
- без месячной аналитики
- без сложных графиков
- без resize блока мышкой по шкале
- drag and drop нужен только для перемещения блоков по дням и внутри дня
- проект должен быть рассчитан на 1–2 дня разработки

### Результат шага

После завершения этого шага должен быть зафиксирован технический фундамент проекта:

- выбран и утверждён стек
- утверждён подход к хранению данных
- утверждены ключевые библиотеки
- зафиксированы ограничения MVP
