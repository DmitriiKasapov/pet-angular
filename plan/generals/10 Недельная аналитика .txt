## Шаг 10. Недельная аналитика

Нужно реализовать отдельную страницу недельной аналитики, на которой пользователь сможет посмотреть, сколько времени было отработано за выбранную неделю, как это время распределилось по дням, проектам и задачам.

Маршрут страницы:

`/analytics`

Эта страница нужна не для редактирования данных, а для просмотра и анализа уже сохранённых worklog-записей.

---

## Назначение страницы

Страница недельной аналитики нужна для:

- просмотра общего количества часов за выбранную неделю
- просмотра распределения часов по дням недели
- просмотра распределения часов по проектам
- просмотра распределения часов по задачам
- быстрого понимания, на что ушло рабочее время

---

## Общая структура страницы

На странице нужно показать:

- заголовок
- переключение недели
- общий итог по часам за неделю
- блок аналитики по дням
- блок аналитики по проектам
- блок аналитики по задачам

Страница должна быть компактной и понятной.  
Не нужно превращать её в тяжёлый dashboard.

---

## Верхняя панель страницы

В верхней части страницы нужно разместить элементы управления неделей.

### Что должно быть в верхней панели

- заголовок страницы
- отображение выбранной недели
- кнопка перехода к предыдущей неделе
- кнопка перехода к следующей неделе
- кнопка возврата к текущей неделе

### Поведение

- аналитика всегда строится только для одной выбранной недели
- при смене недели все данные на странице пересчитываются
- неделя должна определяться как диапазон с понедельника по воскресенье

---

## Общий итог недели

В верхней части аналитики нужно показать суммарное количество часов за выбранную неделю.

### Что должно быть видно

- общий итог в часах
- при желании — количество worklog-записей за неделю

### Пример

- `Всего за неделю: 18 ч`
- `Записей: 7`

---

## Аналитика по дням

Нужно показать, сколько часов было отработано в каждый день недели.

### Что должно быть видно

Для каждого дня:

- название дня
- дата
- суммарное количество часов

### Дни недели

- понедельник
- вторник
- среда
- четверг
- пятница
- суббота
- воскресенье

### Важно

Нужно показывать все 7 дней недели, даже если в некоторые дни записей нет.

### Если записей нет

Для такого дня нужно показывать `0 ч`.

---

## Аналитика по проектам

Нужно показать, сколько времени за выбранную неделю ушло на каждый проект.

### Что должно быть видно

Для каждого проекта:

- код проекта
- название проекта
- суммарное количество часов

### Правило расчёта

1. взять все worklog-записи выбранной недели
2. определить задачи этих записей
3. определить проекты этих задач
4. суммировать `durationHours` по каждому проекту

### Допустимое поведение

Можно отсортировать проекты по убыванию количества часов, чтобы самые загруженные были сверху.

---

## Аналитика по задачам

Нужно показать, сколько времени за выбранную неделю ушло на каждую задачу.

### Что должно быть видно

Для каждой задачи:

- код задачи
- название задачи
- статус
- проект
- суммарное количество часов

### Правило расчёта

Суммируются все `durationHours` у worklog-записей выбранной недели, связанных с конкретной задачей.

### Допустимое поведение

Можно отсортировать задачи по убыванию количества часов.

---

## Формат отображения

Для MVP достаточно простого табличного или списочного представления.

### Допустимые варианты

- таблицы
- карточки
- простые секции со строками

### Не требуется

- сложные диаграммы
- графики
- canvas-визуализации
- внешние chart-библиотеки

---

## Данные для аналитики

Для построения аналитики страница должна использовать:

- worklog-записи
- задачи
- проекты

### Что нужно вычислять

Для выбранной недели нужно вычислять:

- общий итог часов
- часы по дням недели
- часы по проектам
- часы по задачам

Эти данные нужно готовить через сервис, а не собирать вручную в шаблоне.

---

## Поведение при отсутствии данных

Если за выбранную неделю нет ни одной записи, нужно показать корректное пустое состояние.

### Пустое состояние должно содержать

- понятный текст
- общий итог `0 ч`
- пустые или нулевые значения по дням
- сообщение, что за выбранную неделю записей нет

---

## Рекомендуемая структура страницы

Страница может состоять из следующих блоков:

- `page-header`
- `week-switcher`
- `weekly-total-card`
- `weekly-days-summary`
- `weekly-projects-summary`
- `weekly-tasks-summary`
- `empty-state`

---

## Что не нужно делать на этом шаге

- не добавлять месячную аналитику
- не добавлять аналитику по пользователям
- не добавлять графики
- не добавлять экспорт
- не добавлять фильтры по статусам
- не смешивать недельную аналитику со страницей проекта

---

## Ожидаемый результат шага

После завершения этого шага должно быть реализовано:

- страница `/analytics`
- переключение недели
- общий итог часов за неделю
- аналитика по 7 дням недели
- аналитика по проектам
- аналитика по задачам
- корректное пустое состояние для недели без записей
